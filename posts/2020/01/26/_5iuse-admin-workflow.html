5iuse-admin项目工作流程介绍。

# 后端流程

后端项目仓库：5iuse-admin

# 分支

  • dev：后端默认在dev分支开发，可以提交完成一半的代码，不用提交5iuse-ui文件。
  • master：当要发版时合并到master分支，并确保程序可以运行。

注意： 项目自动生成的5iuse-ui代码请提交到前端仓库5iuse的generate分支。






 
 





# 平时在dev分支开发
# TODO
# 发布时切换到master分支
$ git checkout master
# 合并dev分支
$ git merge dev
# 如果有冲突请解决冲突后提交，没有冲突直接push
$ git add .
$ git commit -m '解决冲突'
$ git push origin master

在master分支提交是会触发github actions自动构建，所以务必保证master分支上的代码是可运行的！

最正规的做法是写好测试用例，然后在Dockerfile里如果不通过就不会构建。

# 前端流程

前端项目仓库：5iuse

# 分支

  • generate：后端提交项目自动生成的5iuse-ui代码。
  • dev：前端默认在dev分支下开发，但是要先合并generate分支的代码。
  • master：当要发版时合并到master分支。







 



 





# 平时在dev分支
# TODO
# 后端更新了generate分支代码，则合并
$ git checkout generate
$ git pull origin generate
$ git checkout dev
$ git merge generate
# 完成开发，要发布版本时，合并到master分支
$ git checkout master
$ git merge dev
# 如果有冲突请解决冲突后提交
$ git add .
$ git commit -m '解决冲突'
$ git push origin master

同样可以自动触发前端构建，自动更新。
